3. What is event bubble?

One behavior these events have in common is Event Bubbling. I'll explain what this behavior meansEvent Bubbling is a concept in the DOM (Document Object Model). It happens when an element receives an event, and that event bubbles up (or you can say is transmitted or propagated) to its parent and ancestor elements in the DOM tree until it gets to the root element.The "Event Bubbling" behavior makes it possible for you to handle an event in a parent element instead of the actual element that received the event.

4. What is a callback function and why will you use it?

The main purpose of a callback function is to allow the calling function to call back or notify the caller that some task has been completed or some condition has been met, without the caller having to constantly poll or check for that condition. Callback functions are commonly used in event-driven programming, where they are passed to event handlers to be called when a certain event occurs. They are also used in asynchronous programming, where they are passed to asynchronous functions to be called when the asynchronous operation has completed.
